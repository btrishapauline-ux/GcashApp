Sa enum TransactionType ginagamit to para ma classify ang bawat transaction sa history and ang mga variable na ginamit ko rito will remain constant. Sa AccountStatus naman ay para sa pag secure ng account. active pag online ang user, locked pag nag exceed sa tatlong try ang user sa pag enter ng pin, and suspend naman pag hindi na ginagamit ang account. Sa Transaction class naman nakalagay ang mga records ng transactions and ang remaining balance ng account ng user and sa reference ito ay palaging unique sa bawat transaction dahil ito ay di pwede magkaroon ng kapareho. Sa GCashAccount ko naman nilagay ang abstract eto yung main class ng Gclass account, dito naka store ang information and functionality ng account. BasicAccount naman ang mapupuntahan ng user pag pinindot niya ang unverified na account where in mas mababa ang limitations ng pag send and receive ng user compared sa verified account. In the contrary, nasa VerifiedAccount Class naman ang mga matataas na limit ng pag transact ng user. si GcashSystem class naman ang nag mamanage ng accounts at transactions, eto yung nagsisilbing core system ng applications. Sa mga sumunod na code ay mga functions ng bawat features and lastly, Input validation helpers. Dito nilalagay yung mga bawal or anong tamang ilagay na value sa bawat variables.
